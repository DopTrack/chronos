TU Delft Styleguide

This is a automated styleguide for the TU Delft. The style guide consists of two parts. One part are html templates of several TU Delft webpages. The other part is documentation about all the elements used in those templates, for example the different types of buttons, headers, icons etc.

The templates are generated using Middleman, which also doubles as the webserver. The documentation with examples is generated with Hologram

Some template code and documentation code is shared, so that adjustments in code partials are updated in both the templates and de docs. This is to make development easier, and allows you to manage your code in one place. So don’t code in the templates. Update via the partials as much as possible!

Installation

You will need a ruby capable environment. Visit the ruby site to view installation instructions on how to get up and running with ruby.

Once Ruby is installed, we need to install Bundler by typing in a terminal gem install bundler. Bundler is an app that manages the version and the dependencies of your software stack. It allows us to install all the other software needed to run this guide in one line in your terminal. Also, it will help to update old software, and rollback if a newer version breaks anything.

The repository contains the configuration for Bundler to use (it refers to the file Gemfile). So no actions are necessary to configure Bundler.

Once bundler is installed, you can install the neccesary software by typing the following in your terminal: bundle install. This will install all the required software, such as Middleman.

Running

Assuming your install went correctly, you can run the app by executing bundle exec middleman in your terminal. Follow the instructions and visit http://localhost:4567 once the app is running to view the index page (home).

Once it's running, Hologram makes a new build of the docs http://localhost:4567/docs available.

Note: To build manually, remove activate :hologram on line 27 in the config.rb, and type in bundle exec hologram in the terminal every time you've made changes to the source and need to see the updated version.

Updating components

To illustrate the above, we’ll describe an example of the steps you need to take when updating an element, let’s say the accordion component. If we're not sure where we can find a certain component, we can look it up in either the docs (where an url is located beneath the header of each component), or by using the element inspector in chrome. Classnames should correspond with filenames. Since the classname is .accordion, we can find the haml/html partial in source/components/_accordion.haml, the (s)css in source/css/project/components/_accordion.scss, and the javascript in source/js/_accordion.js. Update what you need and when you are done run bundle exec hologram. This builds the documentation in the source folder so you can see the changes.

Creating components

Let's assume you want to add a new component to the library. First you need to create a HAML file in the source/components/ folder. Let's call this file _foobar.haml. We use an underscore here because we only need to include this file, and not export it. You fill this partial with some html and hit save. You can now use it in a template by typing =partial 'foobar' in the appropriate file (created or found in source). To style it, and to make it show up in the docs, also create a _foobar.scss in the source/css/project/components folder and add the following comment:

/*doc
---
title: foobar
category: component - foobar
---

Just a simple foobar.

```component_example
filename: foobar
```

*/

This will cause hologram to recognize the file as a component, and have it available in the docs.

Notes on updating

If you want to update the docs, make sure to create/update the partials and not write raw html in the docs unless it's a global style. The reason is that you want a single source to build the site with, and not have to update html in two different places.

Project Structure

folder used by function
data middleman, hologram data folder used by middleman to populate partials
doc_source hologram files used by hologram to generate the docs. Of special note is markupexampletemplate which is used for the 'example' and 'code' tabs
icons middleman, hologram this houses all the icons. If you want new icons place an svg here and it will take care of itself
lib middleman, hologram used by middleman to autobuild the docs, and by hologram to render the view partials
source middleman the project source as it appears in the browser. You'll spend most of your time here
source/components middleman, hologram All the different building blocks of the site are here, written in HAML.
source/css hologram You can find all of the css here. Don't touch the files in the generated subfolder. These are generated using external programmes, and you will lose your data if you tinker here.
source/docs hologram This is the folder hologram copies the files from it's build to
source/fonts hologram The HelveticaNeue and Icon fonts
